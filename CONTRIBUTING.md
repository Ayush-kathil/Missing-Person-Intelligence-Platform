# Contributing to AI Surveillance System

First off, thank you for considering contributing to the **AI Surveillance System**! :tada: It's people like you that make GSSOC such a great community.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. We expect all contributors to maintain a welcoming and inclusive environment.

## Getting Started

1. **Fork the repository**: Click the "Fork" button in the top right corner of the repository page.
2. **Clone your fork**: `git clone https://github.com/<your-username>/ai-surveillance-system.git`
3. **Create a branch**: `git checkout -b feature/your-feature-name` or `git checkout -b bugfix/your-bug-name`
4. **Make your changes**: Implement your feature or bug fix.
5. **Commit your changes**: Write clear, descriptive commit messages.
6. **Push to your fork**: `git push origin your-branch-name`
7. **Submit a Pull Request**: Open a PR against the `main` branch of this repository.

## Issues

- **Search first**: Before opening a new issue, please check if a similar issue already exists.
- **Use templates**: When opening an issue, please use the provided templates (Bug Report or Feature Request) and fill in as much detail as possible.
- **Claiming issues**: If you want to work on an issue, please leave a comment asking to be assigned. **Do not start working on an issue until you have been assigned to it.**

## Pull Requests

- **Link to issue**: Every PR should be linked to an open issue.
- **Describe your changes**: Clearly explain what you have changed and why.
- **Testing**: Ensure that your changes do not break any existing functionality. If you added new features, add appropriate tests if applicable.

We look forward to reviewing your contributions! :rocket:
